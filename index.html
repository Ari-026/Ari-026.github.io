プロジェクト概要

今回の卒業制作では、3つの異なるシステム開発に挑戦する予定です。

  • バイト先の在庫管理システム
  • 個人向け**予定管理アプリ**
  • **インフラ監視・管理システム**

各プロジェクトで異なる技術スタックを経験し、実践的なスキルを習得します。

1. バイト先向け在庫管理システム

手作業に頼りがちな在庫管理の非効率性を解消し、店舗運営を最適化します。

主要機能

商品・在庫登録

商品情報と現在の在庫数を一元管理します。

入出庫管理

リアルタイムでの在庫変動を記録・反映します。

発注点アラート

在庫不足を自動で通知し、欠品を防止します。

発注リスト自動生成

効率的な発注業務を支援します。

在庫レポート

在庫状況や回転率を可視化します。

技術要素

バックエンド

Python (Django / Flask)

データベース

PostgreSQL / MySQL

フロントエンド

HTML, CSS, JavaScript (React / Vue.js)

インフラ

AWS / GCP (EC2, RDSなど)

開発ツール

Git / GitHub, Docker

2. 個人向け予定管理アプリ

ユーザーの日常生活や業務をサポートし、効率的な時間管理とタスク遂行を可能にします。

主要機能

スケジュール管理

イベント登録、カレンダー表示で予定を視覚化します。

タスク管理

To-Doリスト、進捗管理、リマインダーを設定します。

通知機能

予定やタスクのリマインドを自動通知します。

共有機能

他のユーザーと予定やタスクを共有できます（任意）。

クロスプラットフォーム

ウェブ、モバイル（iOS/Android）で利用可能です。

技術要素

バックエンド

Node.js (Express) / Python (FastAPI)

データベース

MongoDB / Firestore

フロントエンド (Web)

HTML, CSS, JavaScript (React / Next.js)

フロントエンド (Mobile)

React Native / Flutter

認証

Firebase Auth / Auth0

3. インフラ監視・管理システム

ITインフラの安定稼働を支え、問題の早期発見と対応を自動化・効率化します。

主要機能

サーバー稼働監視

CPU、メモリ、ディスク使用率などをリアルタイムで監視します。

ログ収集・分析

サーバーやアプリログを一元管理し、異常を検知します。

アラート通知

異常発生時、メールやSlackへ自動通知します。

リソース管理

仮想マシンやコンテナのリソース利用を可視化します。

デプロイ自動化

アプリケーションのデプロイプロセスを自動化します。

技術要素

言語

Python / Go

監視ツール

Prometheus, Grafana, Zabbix

ログ管理

ELK Stack (Elasticsearch, Logstash, Kibana)

コンテナ技術

Docker, Kubernetes

構成管理

Ansible / Terraform

クラウド

AWS / GCP / Azure (Monitoring, Logging)

まとめ

これらのプロジェクトを通じて、私たちは以下の目標を達成します。

  • **実践的な開発スキル**: バックエンド、フロントエンド、インフラまで一貫した開発経験
  • **課題解決能力**: 現実世界の課題に対するシステムソリューションの提案と実装
  • **最新技術の習得**: 各分野で広く使われる技術やフレームワークの深い理解
  • **チーム開発の経験**: Git/GitHubを活用した協調開発（チーム制作の場合）

これらのシステム開発が、私たちの卒業制作の集大成となることを目指します。
