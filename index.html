卒業制作

概要

三浦花音

プロジェクト概要

今回の卒業制作では、以下の3つの中から挑戦します。

  • バイト先の在庫管理システム
  • 予定管理アプリの開発
  • インフラ監視・管理システム

1. バイト先の在庫管理システム

手作業に頼っている在庫管理の非効率性を解消

主要機能

商品・在庫登録

商品情報と現在の在庫数を一元管理

入出庫管理

リアルタイムで在庫の変動を記録・反映/p>

発注点アラート

在庫不足を自動で通知し、欠品を防止

発注リスト自動生成

効率的な発注業務を支援

在庫レポート

在庫状況や回転率を可視化

技術要素

バックエンド

Python (Django / Flask)

データベース

PostgreSQL / MySQL

フロントエンド

HTML, CSS, JavaScript (React / Vue.js)

インフラ

AWS / GCP (EC2, RDSなど)

開発ツール

Git / GitHub, Docker

2. 個人向け予定管理アプリ

日常生活や業務をサポートし、効率的な時間管理とタスク遂行を可能にします。

主要機能

スケジュール管理

イベント登録、カレンダー表示で予定を可視化

タスク管理

To-Doリスト、進捗管理、リマインダーを設定

通知機能

予定やタスクのリマインドを自動通知

共有機能

他のユーザーと予定やタスクを共有（任意）。

クロスプラットフォーム

ウェブ、モバイル（iOS/Android）で利用可能にする。

技術要素

バックエンド

Node.js (Express) / Python (FastAPI)

データベース

MongoDB / Firestore

フロントエンド (Web)

HTML, CSS, JavaScript (React / Next.js)

フロントエンド (Mobile)

React Native / Flutter

認証

Firebase Auth / Auth0

3. インフラ監視・管理システム

ITインフラの安定稼働を支え、問題の早期発見と対応を自動化・効率化します。

主要機能

サーバー稼働監視

CPU、メモリ、ディスク使用率などをリアルタイムで監視。

ログ収集・分析

サーバーやアプリログを一元管理し、異常を検知

アラート通知

異常発生時、メールやSlackへ自動通知

リソース管理

仮想マシンやコンテナのリソース利用を可視化

デプロイ自動化

アプリケーションのデプロイプロセスを自動化

技術要素

言語

Python / Go

監視ツール

Prometheus, Grafana, Zabbix

ログ管理

ELK Stack (Elasticsearch, Logstash, Kibana)

コンテナ技術

Docker, Kubernetes

構成管理

Ansible / Terraform

クラウド

AWS / GCP / Azure (Monitoring, Logging)

まとめ

これらのプロジェクトを通じて、以下の目標を達成したいと考えています。

  • 課題解決: 現実世界の課題に対するシステムソリューションの提案と実装、実用化を目指します。
  • 最新技術の習得: 各分野で広く使われる技術やフレームワークの深い理解を心がける。

これらのシステム開発を卒業制作とする予定です！
